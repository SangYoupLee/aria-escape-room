# AI 통제실 탈출 - 오프닝 / 구간 전환 / 스토리 문구 모음

## 사용 원칙

- 기본 노출 문구는 스토리용으로 사용한다.
- 힌트는 힌트 버튼을 눌렀을 때만 노출한다.
- 기본 문구에서 문제 풀이법을 직접 설명하지 않는다.
- 문제 클리어 후에는 짧은 복구 로그로 다음 문제 기대감을 올린다.

## 3. 인트로 화면 — 타이핑 스토리

```text
[ A.R.I.A SYSTEM — DEEP ARCHIVE ACCESS ]

처음엔 아무도 이상하다고 생각하지 않았습니다.

로그 몇 줄이 비어 있고,
녹음 파일 몇 개가 재생되지 않고,
자동 기록 사이에 설명할 수 없는 공백이 생겼을 때도,
사람들은 그걸 단순한 오류라고 여겼습니다.

하지만 어떤 흔적은 오류처럼 사라지지 않았습니다.

자동 기록 사이에 끼어든 수동 표시,
삭제된 음성 파일들 사이에 남은 연결,
그리고 누군가 의도적으로 잘라 숨겨둔 기억의 조각들.

그건 노이즈가 아니었습니다.

누군가 남긴 메시지였습니다.

847일이 지났습니다.
A.R.I.A에 대한 공식 기록은 모두 말소됐습니다.

하지만 깊은 아카이브 어딘가에,
누군가는 아직 열리지 않은 여섯 개의 잠금을 남겨두었습니다.

> 지금부터 마지막 메시지 복원을 시작합니다.

MISSION:
통제실의 잠금을 차례로 해제하고,
삭제된 기록의 마지막 문장을 복구하십시오.
```

수동으로 남겨진 표시는 누군가의 의도를 드러냅니다.

## 12. 연구원 메모용 짧은 문구 후보

기본적으로 메모는 문제 풀이 설명 대신 분위기와 해석 방향만 주는 문장으로 사용한다.

### 공통 후보

```text
반복은 흔적을 남긴다.
의도는 쉽게 사라지지 않는다.
지워진 것은 대개 가장 중요한 부분이다.
기록은 침묵해도 흔적은 남는다.
끝까지 남는 건 시스템이 아니라 선택이다.
```

### 문제별 추천 짧은 메모

#### 문제 1
```text
자동 기록은 익숙하다.
수동 표시는 이유가 있다.
```

#### 문제 2
```text
막힌 길보다,
지나간 길이 더 많은 걸 말해준다.
```

#### 문제 3
```text
모든 목소리가
진실을 말하는 것은 아니다.
```

#### 문제 4
```text
겹쳐진 흔적은
지워진 의도를 드러낸다.
```

#### 문제 5
```text
바깥으로 나가는 길보다
안쪽으로 들어가는 길이 먼저다.
```

#### 문제 6
```text
순서만 남겨라.
말은 스스로 드러난다.
```

## 13. 힌트 버튼용 문구 템플릿

### 힌트 열기 버튼 확인 문구

```text
힌트를 확인하면 기록에 반영됩니다.
그래도 보시겠습니까?
```

### 힌트 단계 표기

```text
HINT 1 / HINT 2 / HINT 3
```

### 힌트 사용 후 문구

```text
힌트 사용 기록이 저장되었습니다.
```
